\# Contributing



Getting started

\- Fork the repo and create a branch from main.

\- Branch naming: feature/<name>, fix/<name>, chore/<name>.



Development

\- Use dotnet restore, dotnet build, dotnet test locally.

\- Add unit tests for bug fixes or new functionality.



Pull Requests

\- Include a clear description, linked issue (if any), and summary of changes.

\- Ensure CI passes and add one reviewer.



Coding style

\- Follow consistent C# idioms, nullability annotations, and XML comments for public endpoints.

\- Keep methods small and well tested.



Issue reporting

\- Provide reproducible steps, expected vs actual, and environment details.



Spekit content

\- If adding Spekit Speks, provide seed json/csv under Presentation/wwwroot/js and update mapping file.
